Automated Testing (Browser Console)

Run automated tests in the browser console:

  1. Open the main site in a new tab
  2. Open Developer Tools (F12 or Cmd+Option+I)
  3. Go to the Console tab
  4. Copy and paste the test script from assets/js/volumetric-clouds/test-scroll-behavior.js
  5. Run: runAllTests()

Or use this shortcut:

// Copy this into the console on the main site
fetch('/js/volumetric-clouds/test-scroll-behavior.js')
  .then(r => r.text())
  .then(code => {
    eval(code);
    runAllTests();
  });

Manual Testing Checklist

Desktop Scroll Behavior (14.1)

  • Scroll down accumulation works (150px threshold)
  • Scroll up stagger delay works (300ms)
  • Quick scroll up is ignored (< 300ms)
  • Scroll indicator shows progress correctly
  • Transitions are smooth (no jank)
  • No console errors

Mobile Touch Behavior (14.2)

  • Swipe up transitions to site (150px)
  • Swipe down at top returns to clouds (300ms delay)
  • Touch gestures feel responsive
  • Performance is acceptable (>45 FPS)
  • Tested on iOS device
  • Tested on Android device
  • No touch event conflicts

Keyboard Navigation (14.3)

  • Arrow Down works in CLOUD_MODE
  • Page Down works in CLOUD_MODE
  • Space works in CLOUD_MODE
  • Arrow Up works at top in SITE_MODE
  • No keyboard traps
  • Focus management is correct
  • Can navigate freely between modes

Test Scenarios

Scenario 1: First-time Visitor

  1. Open the main site
  2. Verify clouds appear immediately
  3. Scroll down to explore the site
  4. Verify smooth transition to site content

Scenario 2: Returning to Clouds

  1. From site content, scroll to the top
  2. Continue scrolling up for 300ms
  3. Verify clouds reappear
  4. Verify smooth transition

Scenario 3: Keyboard User

  1. Start at clouds
  2. Press Arrow Down to enter site
  3. Use Tab to navigate content
  4. Press Arrow Up at top to return to clouds
  5. Verify no keyboard traps

Scenario 4: Mobile User

  1. Open site on mobile device
  2. Swipe up to enter site
  3. Swipe down at top to return to clouds
  4. Verify smooth performance

Debugging Tools

Use these commands in the browser console on the main site:

// Check current state
console.log('State:', window.scrollManager.state)
console.log('Scroll Y:', window.scrollY)
console.log('Accumulator:', window.scrollManager.scrollAccumulator)

// Force state changes
window.scrollManager.transitionToCloud()
window.scrollManager.transitionToSite()

// Monitor events
window.addEventListener('wheel', (e) => {
    console.log('Wheel:', e.deltaY, 'State:', window.scrollManager.state)
})
